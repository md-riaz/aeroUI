Project documentation

This guide shows how to build shadcn-inspired interfaces with Tailwind utility classes using the aeroUI source file. Copy the markup you need, add src/aeroui.css to your Tailwind build, and compile a stylesheet that fits your project.

Remember the motto: shadcn UI principles layered on Tailwind CSS, paired with the semantics and interactive plugins from Bootstrap so the entire ecosystem feels cohesive.

Setup

aeroUI ships as Tailwind source. Copy src/aeroui.css into your project, register it with your Tailwind configuration, and let your build tool emit the final CSS your app will ship. Keep bootstrap.bundle.min.js and a lightweight jQuery include in place so existing JavaScript interactions continue to fire.

1. Add aeroUI to Tailwind

Copy the source file into your project (for example resources/css/aeroui.css) and include it in tailwind.config.js so the compiler can see your component templates. 

// tailwind.config.js
module.exports = {
  content: [
    "./resources/views/**/*.blade.php",
    "./public/**/*.html",
  ],
  theme: { extend: {} },
  plugins: [],
};

// resources/css/app.css
@import "./aeroui.css";

2. Compile your stylesheet

Run the Tailwind CLI, Vite, Mix, or your preferred build tool to generate the CSS that applies aeroUI’s utility recipes to your markup. 

npx tailwindcss -i ./resources/css/app.css -o ./public/build/aeroui.css --minify

3. Load your compiled stylesheet

Reference the CSS emitted by your build as the primary stylesheet for your project. aeroUI is designed to replace legacy Bootstrap styles with Tailwind utility classes, so you no longer need to ship Bootstrap’s CSS alongside it. The docs/assets/aeroui.css file that powers this site is a demo artifact, not a published distribution. 

<link rel="stylesheet" href="/build/aeroui.css">

4. Keep Bootstrap JS for interactivity

While aeroUI replaces the Bootstrap CSS layer, its interactive components still rely on Bootstrap’s JavaScript bundle and a light sprinkle of jQuery helpers. Ship bootstrap.bundle.min.js alongside jQuery so data attributes, toggles, and other behaviours continue working. 

<script src="https://cdn.jsdelivr.net/npm/jquery@3.7.1/dist/jquery.min.js"></script>
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"></script>

Component reference

Browse blocks

Browse src/components for reusable UI such as headers, pricing sections, forms, and dashboards. Each snippet drops into your Tailwind-powered templates, and your build picks up the selectors automatically.

Utilities like text-muted-foreground or bg-card are authored in Tailwind but compiled into the CSS your build emits. Layer them with your own utility classes for layout tweaks.

When you need to compose more complex experiences, stack blocks together—hero, feature grid, pricing, testimonials—to create a shadcn-inspired landing page without hand-writing extra CSS.

Customising aeroUI

Rebuild the CSS

Duplicate tailwind.config.js, tweak colours or spacing, then run npm run build. Point the output to your project’s public directory (for example public/build/aeroui.css).

Extend tokens

Add custom CSS variables or override existing ones by layering a small stylesheet after aeroUI. Because everything compiles to CSS variables, you can change palettes without editing every component.

Maintain consistency

Commit the generated CSS your build produces alongside your project to keep deployments deterministic. Replace the file when new releases ship.

Dark mode ready

Toggle the data-theme attribute to instantly switch between light and dark palettes without additional setup.

Keeping aeroUI updated

Because aeroUI is file-based, updates are as simple as pulling the latest src/aeroui.css, re-running your Tailwind build, and copying new snippets into your project.

For version control, commit the stylesheet to your repository so that deployments stay deterministic.
