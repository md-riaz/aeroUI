Project documentation

This guide shows how to integrate aeroUI into an existing Bootstrap build without a package manager. Every snippet in the repository is copy-ready, and the compiled CSS is all you need to ship production interfaces.

Need live sections? Visit the Blocks directory for marketing layouts you can paste into your pages.

Setup

aeroUI is distributed as vanilla HTML and a single CSS file. Place dist/aeroui.css in your project and reference it after Bootstrap’s stylesheet.

1. Include the CSS

Insert the aeroUI stylesheet after Bootstrap. The cascade ensures our tokens override the defaults. 

<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css">
<link rel="stylesheet" href="/assets/css/aeroui.css">

2. Paste markup

Copy components from src/ or the demo files. The markup is standard Bootstrap 5 so you can use existing grid, form, and utility classes alongside aeroUI tokens.

3. Enable JavaScript behaviour (optional)

Components that rely on Bootstrap interactivity, such as dropdowns or tabs, only need the Bootstrap bundle script. No npm scripts or bundlers are necessary. 

<script defer src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"></script>

Component reference

Browse src/components for reusable UI such as headers, pricing sections, and forms. Each snippet is designed to be copied into Bootstrap templates without additional build steps.

Utilities like text-muted-foreground or bg-card are authored in Tailwind but compiled into the distributed CSS. You can layer them with Bootstrap utilities for layout tweaks.

When you need to customise tokens (for example, changing brand colours), duplicate the Tailwind config and rebuild the CSS with npm run build. The distributed CSS keeps working even if you skip this step.

Keeping aeroUI updated

Because aeroUI is file-based, updates are as simple as replacing dist/aeroui.css with the latest release and copying new snippets into your project.

For version control, commit the stylesheet to your repository so that deployments stay deterministic.

Watching for changes? Star the GitHub repository to receive update notifications.
